# Asset brief — Fallen-log cover set

Implements backlog idea #3 in [../park_asset_ideas.md](../park_asset_ideas.md):
natural low cover that fits Bouneurmaum National Park (pine forest, road
approaches, forest edges) far better than generic barricades. Setting reference:
[../../setting.md](../../setting.md).

## Goal

A small family of weathered fallen-pine logs usable as readable, jumpable low
cover. Built to real meters, exact mesh collision, sharing one set of materials.

## Variants (4)

| Prefab | Model | Role |
|--------|-------|------|
| `FallenLogStraight` | `fallen_log_straight.vmdl` | Long straight trunk; baseline cover (~7 m). |
| `FallenLogAngled` | `fallen_log_angled.vmdl` | Leaning ~30°, one end raised; cover with a step-up silhouette. |
| `FallenLogHollow` | `fallen_log_hollow.vmdl` | Tube with a bore through it; crawl/peek-through cover. |
| `FallenLogMossy` | `fallen_log_mossy.vmdl` | Straight trunk with moss on the upper bark; damp forest dressing + cover. |

The backlog's "branchy" variant is folded into the weathered styling (broken
branch stubs on every log) rather than shipped as a 5th prop.

## Style

Weathered fallen pine: tapered trunk, slight bend, one sawn-flat end and one
broken/jagged end showing growth rings, 2–3 broken branch stubs, low-frequency
bark surface noise, occasional bark-loss facets.

## Materials (shared, `Assets/materials/environment/`)

- `fallen_log_bark.vmat` — brown furrowed bark (sides). metalness 0, rough 0.88.
- `fallen_log_endgrain.vmat` — concentric growth rings (cut/broken ends, stubs,
  hollow interior). metalness 0, rough 0.80.
- `fallen_log_moss.vmat` — mottled moss (mossy variant top faces). metalness 0,
  rough 0.92.

Textures are generated by `scripts/generate_fallen_log_textures.py` (numpy+PIL,
tileable, IDAT chunked ≤64 KB so s&box decodes them — see the log_cabin grey-
texture bug). Blender material slot names must match the remap keys exactly:
`Bark`, `Endgrain`, `Moss`.

## Pipeline / scale / collision

- Configs: `scripts/fallen_log_<variant>_asset_pipeline.json`
  (`global_scale 0.0254`, axis `-Y` fwd / `Z` up, `render_mesh` collision,
  `surface_prop wood`).
- Model in real meters along Blender **+X**, bottom at **Z=0** (rests on flat
  ground). Prefab `Visual.Scale = 15.5` → real size (39.37 units/m).
- Exact mesh collision: `PhysicsMeshFile` nested in `PhysicsShapeList`; prefab
  has ONE `Static` `ModelCollider` on the `Visual` child (same GameObject as the
  `ModelRenderer`). See [../../collision_authoring.md](../../collision_authoring.md).

## Placement notes

Logs rest on Z=0, so they sit on flat terrain by default; lower/rotate per
instance on slopes. Prefabs appear under **Drone vs Players / Environment /
Fallen Logs** in the create menu.
